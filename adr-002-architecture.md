# ADR-002: Вибір архітектурного стилю — Моноліт
 
## Статус
Accepted
 
## Контекст
Система розробляється для автоматизації роботи диспетчерського центру. На початковому етапі важливо швидко запустити MVP та забезпечити легкість тестування основних бізнес-процесів.
 
## Рішення
Обрати монолітний архітектурний стиль.
 
## Обґрунтування
1. **Швидкість розробки:** Весь код знаходиться в одному репозиторії, що спрощує взаємодію між модулями.
2. **Простота розгортання:** Потрібно підтримувати лише один серверний застосунок.
3. **Низька складність:** Відсутність проблем із мережевими затримками між сервісами.
 
## Альтернативи
- **Мікросервіси:** Відхилено через надмірну складність інфраструктури для даного етапу проєкту.
 
## Наслідки
### Позитивні
- Максимально швидка реалізація логіки.
- Просте наскрізне тестування.
### Негативні
- Складніше масштабувати окремі модулі у далекому майбутньому.
